# Phase 6.8 independent fixture methodology

The Engine-owned Phase 6.8 regression suite is generated by
`crates/secure-engine/tests/remediation_phase68.rs`. It was authored from the seven public invariant
families and general framework semantics. It does not copy Secure Bench source, paths, identifiers,
hashes, or answers.

The balanced matrix contains 56 isolated scenarios: one vulnerable and one safe control for every
combination of seven rule families and four variants. Across the matrix, the variants cover
JavaScript, JSX, TypeScript, and TSX; Node.js, Express, Next.js App Router, and Server Actions; and
direct, helper-mediated, inter-file aliased, and control-flow-sensitive topology.

Each vulnerable scenario must produce exactly one intended finding with exact taxonomy, CWE,
canonical source and sink kinds, expected endpoint spans, a connected realizable path, and no
barrier or uncertainty. Its paired safe control must not emit the family finding. Additional tests
cover:

- identifier renaming and harmless statement insertion;
- helper extraction and return propagation;
- request and form-data destructuring;
- exact fixed arrays, sets, and literal fallbacks;
- unsafe suffix checks and destination blocklists;
- fixed allowlists mutated before use;
- fixed executable argument vectors with default no-shell behavior and explicit `shell: true`;
- dynamic process-option spreads that could enable shell behavior;
- authentication without operation authorization;
- ambiguous aliases, wrong namespace receivers, repository-escaping imports, same-named helpers,
  cycles, and recursion.

The suite also compares repeated semantic fingerprints and retains the Phase 6.7 pinned
fingerprints for unaffected historical fixtures. A cache-envelope change causes an intentional
cold miss and then byte-stable warm results.

This corpus is a deterministic development gate, not a representative population, unseen
evaluation, scanner comparison, or evidence of production readiness or complete coverage.
