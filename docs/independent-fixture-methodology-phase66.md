# Phase 6.6 independent fixture methodology

The Phase 6.6 regression corpus is authored entirely in Secure Engine and is generated by `crates/secure-engine/tests/evidence_semantics_phase66.rs`. It contains 102 isolated scenarios: exactly six vulnerable and six safe scenarios for each `SE1001`–`SE1007` family (84 total), plus 18 Rust, Python, and Go command-semantics variants. JavaScript, JSX, TypeScript, and TSX modes are represented in the family matrix.

Vulnerable cases cover direct flow, assignments and renames, local helpers, aliases/wrappers, inter-file helpers, unrelated guards, and authentication-only mutation guards. Safe cases cover fixed executables with shell disabled, parameterized queries, policy-specific sanitizers, safe parsing without dynamic execution, and operation-specific authorization. Each scenario is scanned in a fresh temporary repository with cache disabled and is asserted against only its intended invariant.

Metamorphic checks require equivalent vulnerable forms to retain one semantic fingerprint across cosmetic renames, harmless insertion, aliasing, and helper extraction. Mutation checks replace an unsafe invariant with its safe counterpart and require the expected finding to disappear; separate guard-removal checks require it to reappear. Import aliases, object destructuring, inter-file wrappers, exact protocol/host/port policies, permissive suffix/substring checks, userinfo checks, path decoding, base joins, and relative-path transformations have focused regressions. Resource checks force a small candidate-path budget and require deterministic truncation plus an explicit limitation.

The corpus and expected outcomes were designed from public security invariants and independently authored source. They contain no Secure Bench case identifier, corpus path, fixture wording, expected result, baseline, evaluator behavior, or scanner-specific exception. Secure Bench private artifacts are neither read nor executed.
