# Phase 6.7 independent fixture methodology

The Engine-owned Phase 6.7 suite is generated by
`crates/secure-engine/tests/generalization_phase67.rs`. It contains exactly 140 isolated scenarios:
ten vulnerable and ten safe scenarios for each of the seven taxonomy families. It was authored from
the public invariant and contract, not from undisclosed benchmark material.

Each family is balanced across five JavaScript and five TypeScript topologies for both vulnerable
and safe cases. Variants cover Node-style handlers, Express-style request collections, Next.js App
Router routes, Server Actions, direct calls, local helpers, wrappers, aliases, inter-file imports,
returns, control flow, renaming, and harmless statement insertion. Safe/vulnerable pairs exercise
guard or sanitizer introduction/removal while holding the surrounding structure stable.

Additional adversarial checks cover:

- filesystem prefix checks without a separator boundary;
- hostname suffix checks and attacker-selected protocol/host comparisons;
- destination blocklists;
- attacker-supplied authorization claims without a trusted principal resolver;
- unrelated sources and fixed sinks;
- ambiguous aliases, recursive and cyclic helpers, malformed syntax, resource bounds, cancellation,
  and host-path privacy.

Every vulnerable scenario must emit exactly one finding for its intended family with contract-v2
source/sink endpoints. Every safe scenario must emit none for that family. Reports must remain below
per-scenario graph bounds and never be truncated. The suite is deterministic local test evidence,
not a representative population or an external benchmark.
