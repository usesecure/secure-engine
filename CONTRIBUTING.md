# Contributing

The current scope accepts changes to repository inventory, supported language adapters, normalized facts, evidence semantics, deterministic rules, suppressions, the local parse cache, public JSON and SARIF contracts, baselines, private local history, optional consent-gated AI validation, native UI boundaries, and Fedora RPM packaging. Automatic fixes, cloud services, telemetry, hosted scanning, and benchmark-specific exceptions remain out of scope.

Run the checks in `docs/development.md` before submitting a change. Contributions are made under the MIT License and must include a Developer Certificate of Origin sign-off (`git commit -s`), certifying that the contributor has the right to submit the work under this license.

Open an issue before large architectural changes. Security reports must follow [SECURITY.md](./SECURITY.md) instead of a public issue.
